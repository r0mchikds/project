# Финальный проект в рамках модуля ML/DL

В этом репозитории собраны два финальных проекта по Kaggle:

1) `Titanic - Machine Learning from Disaster`;
2) `House Prices - Advanced Regression Techniques`;

Проекты выполнялись согласно чек-листу, приведённому к заданию.

## Общая идея

В каждом проекте есть две части:

- `notebook/` — ноутбук с полным исследованием: EDA, preprocessing, кросс-валидация, сравнение моделей, DNN, feature engineering, ансамбли и итоговые выводы;
- `main.py` и `config.py` — воспроизводимый финальный пайплайн для лучшей одиночной модели по проекту.

Дополнительно в корне репозитория находится общий `main.py`, который запускает оба проекта подряд и выводит краткую сводку по результатам.

## Какие модели вынесены в финальные пайплайны

### Titanic

* модель: `RandomForestClassifier`
* версия: до feature engineering

Эта модель показала лучший результат среди одиночных моделей по внутренней `Stratified 5-Fold CV`.

### House Prices

* модель: `CatBoostRegressor`
* версия: после feature engineering

Эта модель оказалась лучшей одиночной моделью как по внутренней `5-Fold CV`, так и по `Public Leaderboard`.

pip install -r requirements.txt

## Запуск

### Запустить оба проекта сразу

```bash
python main.py
```

### Titanic

```bash
python titanic/main.py
```

### House Prices

```bash
python house_prices/main.py
```

После запуска скрипт:

* загружает данные;
* готовит признаки;
* считает метрику на кросс-валидации;
* формирует итоговые предсказания на test через усреднение предсказаний моделей, полученных на фолдах;
* сохраняет сабмит в папку `submissions/`;
* сохраняет JSON с метриками кросс-валидации.

Корневой `main.py` запускает оба проекта подряд и печатает краткую сводку по результатам.

## Основные результаты

### Titanic

| Кандидат               | Модель                            | Mean CV Accuracy |   Public LB |
| ---------------------- | --------------------------------- | ---------------: | ----------: |
| Лучшая модель до FE    | Random Forest                     |       **0.8529** |     0.76794 |
| Лучшая модель после FE | Random Forest                     |           0.8440 |     0.76555 |
| Лучший ансамбль        | Stacking with Logistic Regression |           0.8451 | **0.77990** |

### House Prices

| Кандидат               | Модель                          | Mean CV RMSE |   Public LB |
| ---------------------- | ------------------------------- | -----------: | ----------: |
| Лучшая модель до FE    | CatBoost                        |       0.1241 |     0.12270 |

## Краткие выводы

### Titanic

* Лучшая одиночная модель по внутренней валидации — `RandomForestClassifier` на базовых признаках.
* Feature engineering помог части моделей, но не улучшил лучшую tree-based модель.
* Лучший результат на `Public Leaderboard` дал stacking, а не лучшая одиночная модель.

### House Prices

* Лучшая одиночная модель — `CatBoostRegressor`.
* Переход к `log(SalePrice)` оказался оправдан.
* Feature engineering дал дополнительное улучшение качества.
* Ансамбли не превзошли лучшую одиночную модель после feature engineering.

## Что важно про python-файлы

`main.py` и `config.py` не повторяют весь объём экспериментов из ноутбуков.

Они сделаны для того, чтобы:

* быстро воспроизвести финальный пайплайн;
* получить метрику лучшей одиночной модели;
* получить готовый сабмит.

При этом итоговые test-предсказания в этих скриптах формируются через усреднение предсказаний моделей, полученных на кросс-валидации, как и в ноутбуках.

Полный перебор моделей, DNN и ансамблей сохранён в ноутбуках.
